Detailed Description

Nested loop for IndexScan. This is the implementation of usual nestloop which receives one input table (outer table) and repeatedly does indexscan on another table (inner table) with inner table's index. This executor is faster than HashMatchJoin and MergeJoin if only one of underlying tables has low selectivity.
