Detailed Description

Represents a Temporary Table to store temporary result (final result or intermediate result). Temporary Table has no indexes, constraints and reverting. So, appending tuples to temporary table is much faster than PersistentTable. deleteTuple is not suported in TempTable to make it faster, use deleteAllTuples instead. As there is no deleteTuple, there is no freelist; TempTable does a efficient thing for iterating and deleteAllTuples.
