Get the output number of columns -- strictly for use with plannode classes that "project" a new output schema (vs. passing one up from a child). This is cleaner than using "getOutputSchema().size()" in such cases, such as Projection nodes, when m_outputSchema and m_validOutputColumnCount are known to be valid and in agreement.
