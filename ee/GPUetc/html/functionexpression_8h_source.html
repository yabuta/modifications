00031 // Note that function names and function IDs need not correspond one-to-one.
00032 // A function name (especially one in the Custom/JDBC/OpenGroup category) may be just an alias for another function,
00033 // so it will use the same function ID, e.g. SUBSTR -> SUBSTRING, so SUBSTR does not need its own function id.
00034 // A function name that is "overloaded", applying to different numbers and/or types of arguments may be optimized to map
00035 // to different specialized implementations, each with its own function ID
00036 // e.g. EXTRACT -> either FUNC_EXTRACT_YEAR, or FUNC_EXTRACT_MONTH, or FUNC_EXTRACT_DAY, etc.
00037 // For completeness, FUNC_EXTRACT is defined here, but at this point, it always represents a missing translation step in the parser.
